# クリエイティブ知識ベース 仕様書

## バージョン
- v1.0.0（2026-04-29 初版）

---

## 概要

Meta広告クリエイティブの分析結果を蓄積する知識ベースツール。  
「媒体 × 広告目的 × ジャンル」で分類し、チームで共有・活用する。

### アクセス制御
- Googleアカウントでのログインが必須
- GAS の Script Properties に登録したメールアドレスのみアクセス可
- クライアント招待も同じ仕組みで管理（Gmailアドレスを追加するだけ）

## セットアップ手順

### Step 1: Google Cloud Console でOAuthクライアントIDを取得

1. [Google Cloud Console](https://console.cloud.google.com/) を開く
2. プロジェクトを作成（または既存を使用）
3. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
4. アプリの種類：**ウェブアプリケーション**
5. 承認済みのJavaScript生成元に GitHub PagesのURL を追加  
   例：`https://[ユーザー名].github.io`
6. クライアントIDをコピーしておく（`xxxxxxxxx.apps.googleusercontent.com`の形式）

> ローカル（file://）でテストする場合は `http://localhost` も追加する

### Step 2: Google スプレッドシート + GAS を設定

1. 新しい Google スプレッドシートを作成
2. 「拡張機能」→「Apps Script」を開く
3. `Code.gs` の内容を貼り付けて保存
4. GASエディタで `setup()` 関数を選択して **実行**  
   → シートが初期化され、自分のメールアドレスが許可リストに追加される
5. 「デプロイ」→「新しいデプロイ」→「ウェブアプリ」
   - 実行するユーザー：**自分**
   - アクセスできるユーザー：**Googleアカウントを持つ全員**
6. デプロイURLをコピーしておく（`https://script.google.com/macros/s/…/exec`の形式）

### Step 3: GitHub にアップロード

1. GitHubで新しいリポジトリを作成（名前例：`creative-knowledge-base`）  
   ※ パブリックでも問題なし（データはGASに保存、HTMLにデータは含まれない）
2. `index.html` をリポジトリにアップロード
3. 「Settings」→「Pages」→ Source を `main` ブランチの `/(root)` に設定
4. 数分後に `https://[ユーザー名].github.io/creative-knowledge-base/` でアクセス可能に

### Step 4: 初回セットアップ（ブラウザ）

1. GitHub Pages のURLを開く
2. セットアップ画面が表示されるので以下を入力：
   - **GAS デプロイURL**：Step 2 でコピーしたURL
   - **Google OAuth クライアントID**：Step 1 でコピーしたID
3. 「保存して続ける」→ Google ログイン画面が表示される
4. 自分のGoogleアカウントでログイン → 完了

## アクセス管理（許可リストの操作）

GASエディタから以下の関数を**実行**するだけで管理できる。

```javascript
// メンバー・クライアントを追加
addToAllowlist("member@gmail.com")

// アクセスを削除
removeFromAllowlist("exmember@gmail.com")

// 現在のリストを確認（ログに表示）
getAllowlist()
```

操作後は**再デプロイ不要**（Script Propertiesに即時反映される）。

## 使い方

### 新規登録タブ
月次のMeta広告分析後に入力する。必須項目：分析月・媒体・広告目的・フォーマット・ジャンル・勝ちパターン。

### 知識ベース一覧タブ
蓄積された知見を媒体・目的・タイプ・ジャンル・キーワードでフィルタリング。
誰が登録したかも記録される。

### エクスポートタブ
3形式で出力：
| 形式 | 用途 |
|------|------|
| プロンプト挿入用 | クリエイティブ生成ツールの末尾に貼り付け |
| Markdownドキュメント | 社内共有・議事録用 |
| JSONバックアップ | データのエクスポート |

## データ保存場所

| データ | 保存先 |
|--------|--------|
| 知識ベース（エントリ） | Google Sheets（`entries`シート） |
| 許可リスト | GAS Script Properties |
| GAS URL・クライアントID | 各自のブラウザ localStorage（設定情報のみ） |

---

## 引き継ぎ時の注意

- GASを再デプロイすると**デプロイURLが変わる**。変わった場合は各自の設定を更新する必要あり（ブラウザで「設定」ボタンから更新可能）
- スプレッドシートを削除するとデータが失われる。定期的にJSONエクスポートでバックアップを取ること
